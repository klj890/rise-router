# Rise Router 国际化（i18n）架构

> 版本：v0.1 · 2026-06-13 · 配套：[architecture.md](./architecture.md)、[data-model.md](./data-model.md)

## 0. 四项定调（2026-06-13 用户拍板）

| 决策 | 选定 |
|---|---|
| 覆盖范围 | **全栈**：UI + API 错误 + 通知模板 + 内容数据 |
| 内容数据存储 | **JSONB 本地化字段**（`*_i18n {"zh-CN":…,"en-US":…}`） |
| API 错误本地化 | **后端返回 error code + 结构化参数，消费端按 locale 映射文案** |
| 首批语言 / 默认 | **zh-CN（默认，国情）+ en-US**，架构预留可扩展 |

## 1. 第一性原则：四类本地化彼此解耦

i18n 不是单一机制，而是**四套独立子系统**，混在一起是常见错误：

1. **UI 文案翻译**（前端 i18next 命名空间，key→译文）。
2. **内容数据多语言**（管理员录入的显示名等，存 JSONB）。
3. **API 错误**（稳定 code + 参数，后端不绑定语言）。
4. **格式化**（货币/日期/数字/时区，用 `Intl`/CLDR，与翻译无关）。

它们各自演进、各有 fallback，绝不能用同一张表/同一套 key 硬塞。

## 2. Locale 模型与协商

- 标识：**BCP 47**（`zh-CN`、`en-US`，未来 `zh-TW`/`ja-JP`）。集中配置 `SUPPORTED_LOCALES`、`DEFAULT_LOCALE = zh-CN`，前后端共享同一份常量。
- **协商优先级**（每请求/会话解析一次）：
  1. 显式覆盖：`?lang=` 查询参数 或 `X-Locale` 请求头（调试/分享链接）
  2. 登录用户偏好：`users.locale`
  3. 组织默认：`organizations.default_locale`（白标租户）
  4. `Accept-Language` 头（解析后与支持集匹配）
  5. 系统默认 `zh-CN`
- **fallback 链**（缺 key 时）：`<请求locale> → en-US → zh-CN → 显示 key`（dev）。可配置；`zh-TW→zh-CN`、`en-*→en-US` 等变体回落。
- 前端把选定 locale 持久化（`localStorage: rise-locale`，与主题同模式），并通过 axios 拦截器附 `X-Locale` 头，保证前后端一致；同步设 `<html lang>`。

**时区**与 locale **解耦**：`users.timezone`、`organizations.default_timezone`（默认 `Asia/Shanghai`）。计费/报表的"业务日边界"用**组织级时区**，展示时间戳用用户时区。

**货币**也不由 locale 推导：金额存 `decimal + ISO 货币码`（CNY 默认）；locale 只决定格式（符号位置/分组）。en-US 用户看 CNY 仍是 `¥`，但按 en-US 习惯排版。

## 3. UI 文案（前端）

- 库：**i18next + react-i18next + i18next-browser-languagedetector**（事实标准，支持命名空间、懒加载、ICU 复数）。
- **AntD/dayjs 联动**：locale 切换时同步 `ConfigProvider locale`（zhCN/enUS）与 dayjs locale（当前 `ThemeProvider` 里硬编码的 `zhCN` 改为受控）。
- **Key 策略：命名空间 + 语义 key**（`namespace:key.sub`），**不用** new-api 那种"英文原文当 key"（多团队 + 可插拔平台下，英文文案一改 key 就崩，且 key 绑死一种语言）。
- 命名空间：`common`、`auth`、`errors` + 每个域一套（`pricing`/`billing`/`crm`/`report`/`gateway`/`task`/`identity`/`support`）。
- 资源布局：
  ```
  src/i18n/
    index.ts                 # i18next 初始化、fallback、命名空间注册
    locales/zh-CN/{common,auth,errors,...}.json
    locales/en-US/{...}.json
  ```
  `common`+`auth` 首屏加载；其余按路由懒加载（dynamic import）。
- **可插拔**：MF 插件挂载时 `i18next.addResourceBundle(locale, ns, res)` 注册自己的命名空间（见 §6）。
- 错误文案 = 把后端 `code` 映射到 `errors` 命名空间并插值参数：`t('errors:PRICING_NOT_FOUND', { model, group })`。

## 4. 后端（Rust）

按"API 错误 = code + 参数"的决策，**后端基本不本地化 API 文案**，i18n 职责收窄为：错误契约、通知模板、语言协商。

### 4.1 错误契约（核心）
- `AppError` 携带稳定 `code`（枚举→字符串）+ 可选 `params: serde_json::Value`。
- HTTP body：
  ```json
  { "error": { "code": "PRICING_NOT_FOUND",
               "params": { "model": "gpt-4o", "group": "vip" },
               "message": "未找到定价（zh-CN 兜底，供日志/非 i18n 消费方）" } }
  ```
  `code`+`params` 是**契约正源**；`message` 仅为日志/裸客户端兜底（默认 zh-CN），不作为前端展示来源。
- **error code 是单一正源**：后端枚举 ↔ 前端 `errors` 命名空间一一对应；CI 校验"每个后端 code 都有前端译文"。

### 4.2 通知模板（服务端渲染、按收件人 locale）
- 引擎：**minijinja**（Jinja2 风格、serde 友好、轻量），渲染短信/邮件/站内信。
- 模板注册表键 `(template_key, locale, channel)`；locale 取收件人 `users.locale`（缺则组织默认）。
- 复杂语言能力（性别/复杂复数）暂不需要；**Fluent 作为未来备选**，不提前引入（避免过度设计）。

### 4.3 语言协商中间件 + 服务端小字符串
- `LocaleLayer`（Tower 中间件）按 §2 优先级解析请求 locale，注入 request extension，供通知触发等读取。
- 极少量服务端字符串（如通知主题）用 **rust-i18n**（编译期 `t!` 宏 + JSON）或直接放模板，**不引 Fluent**。

## 5. 内容数据多语言（JSONB 本地化字段）

- 约定：可翻译显示字段存 JSONB，列名后缀 `_i18n`：
  ```json
  "display_name_i18n": { "zh-CN": "企业版", "en-US": "Enterprise" }
  ```
  **稳定 key/slug/code 列保持单语言**（如 `groups.slug`、`models.slug`）——i18n 只覆盖"给人看的名字"。
- 适用字段（跨域）：`models.display_name`、`groups.name`/`description`、套餐/订阅名、`discounts.name`、`datasets.name` 及指标/维度标签、工单分类等。
- **解析与 fallback**：后端 `pick(jsonb, locale)` 按 fallback 链返回最佳串。
  - **读/列表 API**：后端按请求 locale 解析为**纯字符串**返回（payload 小、客户端简单）。
  - **管理编辑 API**：返回**完整 JSONB**（`?i18n=raw`），管理台分语言 Tab 编辑。
- **校验**：至少默认 locale（zh-CN）必须有值；其余可缺（走 fallback）。

## 6. 可插拔 i18n（接入 App Registry，狗粮原则）

App Manifest 增 `i18n` 块：
```yaml
i18n:
  namespaces: [ticket]                                  # 该 App 拥有的命名空间
  locales: [zh-CN, en-US]
  bundleUrl: /plugins/ticket/locales/{locale}/{ns}.json # 前端译文包
  errorCodes: [TICKET_CLOSED, TICKET_ASSIGN_DENIED]     # 该 App 的错误码
  templates: [ticket.assigned, ticket.replied]          # 该 App 的通知模板键
```
- **前端**：Shell 加载插件时按 manifest 拉取并 `addResourceBundle` 注册其命名空间；`GET /api/i18n/manifest?locale=` 返回已启用 App 的命名空间 + bundleUrl，告诉 Shell 该懒加载什么。
- **后端**：App 声明的 error codes / 通知模板合并进各自注册表。
- 内部域与第三方走**同一套注册方式**（狗粮原则）。

## 7. 格式化层（Intl / CLDR）

- 前端封装绑定"当前 locale + tz"的助手：`formatCurrency(amount, currency)`、`formatDate(tsUtc)`、`formatNumber`、`formatRelative`，底层 `Intl.NumberFormat/DateTimeFormat/RelativeTimeFormat/PluralRules`。
- 金额：显式货币码 → `Intl.NumberFormat(locale,{style:'currency',currency})`；**绝不存格式化后的字符串**。
- 日期：存 UTC `timestamptz`，展示按用户/组织时区。
- 复数：需要计数的消息用 ICU MessageFormat（`i18next-icu`），中文无复数、英文有，ICU 统一处理。

## 8. 翻译治理 / 工作流

- 编写基准：zh-CN（国情优先），en-US 镜像；命名空间语义 key。
- CLI `i18n:check`（接入 CI）：① 跨 locale 缺失 key；② 未使用 key；③ 后端 error code 无前端译文。对标 new-api 的 `i18n:sync`，但基于语义 key。
- 暂不接外部 TMS（避免过度设计），JSON 入库 + PR 评审；预留 TMS 导入/导出 seam。
- 可选 dev 伪本地化 `en-XA`（拉长字符串）暴露硬编码文案。

## 9. 数据模型增量（并入 data-model.md）

- `users.locale VARCHAR(16) NULL`、`users.timezone VARCHAR(64) NULL`。
- `organizations.default_locale VARCHAR(16) NULL`、`organizations.default_timezone VARCHAR(64) DEFAULT 'Asia/Shanghai'`。
- 跨域显示字段改 `*_i18n JSONB`（保留 slug/code 单语言）。
- `notification_templates(template_key, locale, channel, subject, body, …)`（通知域，按 locale）。
- error code **不建表**：后端枚举 + 前端 `errors` 命名空间为正源，文档登记。

## 10. 与里程碑的衔接

i18n 是横切关注点，**分散随各里程碑落地**，不集中返工：
- **i18n 基础（建议作为下一个小任务）**：前端 i18next + locale store + AntD/dayjs 联动 + 语言切换控件；后端 `LocaleLayer` + 错误码契约骨架；zh-CN/en-US 基础命名空间（common/auth/errors）。后续每个里程碑**边建边写**自己命名空间的 key 与内容 `_i18n` 字段。
- **通知 i18n** 随通知/财务里程碑落地（minijinja 模板 + `notification_templates`）。
- **每个域**在实现时即采用命名空间 key + `_i18n` 字段，不事后补。

## 11. 技术选型小结

| 层 | 选型 |
|---|---|
| 前端 UI | i18next + react-i18next + i18next-icu；AntD/dayjs locale 联动 |
| 内容数据 | PostgreSQL JSONB `*_i18n` + `pick(jsonb, locale)` |
| API 错误 | error code + params 契约（后端不本地化）；前端 `errors` 命名空间映射 |
| 通知 | minijinja 模板 ×（template_key, locale, channel） |
| 服务端串 | rust-i18n（小集）；Fluent 暂不引 |
| 格式化 | `Intl`/CLDR（货币/日期/数字/复数/时区） |
| 协商 | Tower `LocaleLayer`，优先级 URL>用户>组织>Accept-Language>默认 |
| 可插拔 | App Manifest `i18n` 块 + `addResourceBundle` + `/api/i18n/manifest` |
